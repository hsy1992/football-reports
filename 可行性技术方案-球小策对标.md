# 球小策对标 · 可行性技术方案

> **目标**：基于 [球小策《球小策是如何分析比赛的》](https://www.qiuxiaoce.com/tong-zhi-gong-gao/2026/02/qiu-xiao-zhi-shi-ru-he-fen-xi-bi-sai-de/) 文档，评估其架构在本项目（football-odds-reports）中的可行性，并形成分阶段落地方案。
>
> **评估基调**：本项目已用 98 场赛果 + 194 注模拟下注严格回测，得出三条铁律——① 模型≈市场，理论 edge≈0；② 资金流向/盘口移动与赛果统计上不可区分于噪声；③ 高 EV 注单实测反向（46% 胜率 / -18.7% ROI）。因此**纯赔率维度已到天花板，寻找真实 edge 的唯一方向是赔率之外的信息层**——这正是球小策文档的主轴。方向对，问题只剩"免费/低成本数据能否榨出信号"。

---

## 一、文档解构：球小策在做什么

撇开营销话术（"第三代全息评分""5 个思维线程""独家系数"），其架构本质是一个 **多源数据 + 多因子评分 + 规则仲裁 + 策略组合** 的规则系统（而非真正的 LLM 推理），分 8 层：

| 层 | 名称 | 实质 |
|----|------|------|
| 1 | 数据引擎 | 5 类数据源 -> 200+ 维特征包 |
| 2 | 球员全息评分 | 8 套按位置加权的球员算法评分（0-100） |
| 3 | 市场量化 | 指数四维分析（趋势/波动/异常/阻力）+ 首发预测 |
| 4 | 预处理与风控 | 强制搜索网关 + 一票否决（重伤/轮换降级） |
| 5 | 联赛分级 | 三档联赛，权重优先级不同 |
| 6 | 五维引擎仲裁 | A 硬实力 / B 市场 / C 情报(否决权) / D 概率 / E 终极仲裁 |
| 7 | 战略分流 | 按评分差距分 4 条路径（碾压/胶着/博弈/冷门） |
| 8 | 策略输出 | 单箭头 or 双选组合，6 套策略 + 赛果推演 |

**关键洞察**：文档里"AI 5 个思维线程"= 5 个独立因子打分 + 一个按优先级合并的规则仲裁器；"终极大仲裁"= `信情报 > 信市场 > 信实力` 的优先级链。这是**可复刻的规则系统**，不是黑盒 AI。

## 二、与现有系统的映射：你已经有什么

| 球小策层 | 本项目现状 | 差距 |
|----------|-----------|------|
| 1. 官方赛事数据 | ✅ The Odds API + ESPN 赛程 | 基本齐全 |
| 1. 球员级高阶统计 | ⚠️ FBref 仅用球队级 xG | **未做球员级** |
| 1. 战绩/交锋/积分 | ✅ ESPN 近况 + FBref | 齐全 |
| 1. 市场情绪/成交量 | ❌ 无（且已被自家回测证伪） | **数据不可得 + 已证伪** |
| 1. 24h 指数监控 | ✅ 多帧快照序列 | 齐全（异动捕获可加） |
| 2. 球员评分 | ❌ 无 | 全新模块 |
| 3. 指数四维分析 | ⚠️ 有走势序列，未结构化为四维 | 可扩展 |
| 3. 首发预测 | ❌ 无 | **数据难** |
| 4. 伤停/裁判风控 | ⚠️ 伤停有(懂球帝)，裁判无 | 裁判需新源 |
| 5. 联赛分级 | ❌ 无（仅小组/淘汰赛埋点） | 纯规则，易加 |
| 6. 概率引擎 | ✅ **已有泊松 + Dixon-Coles** | 仅缺蒙特卡洛（一行代码） |
| 6. 多引擎仲裁 | ❌ 单一数学入口 | 全新模块 |
| 7. 战略分流 | ❌ 无 | 纯规则 |
| 8. 策略组合 | ⚠️ 单注 EV，无双选组合 | 可扩展 |
| — 校准/复盘 | ✅ **已有 calibration + paper_bets** | 复用来验证一切新维度 |

**结论**：底座（数据管道、概率模型、校准复盘、自动发布）你已经比球小策文档描述的更扎实。缺的是**信息层**（球员评分、裁判、首发）和**仲裁/分流/组合**这三个规则层。

## 三、逐层可行性评估

图例：✅ 可行（有免费数据） / ⚠️ 部分可行（需新数据源或工程量） / ❌ 阻塞（数据不可得或已证伪）

### 第 1 层 · 数据引擎

| 数据类 | 可行性 | 数据源 / 说明 |
|--------|--------|---------------|
| 官方赛事数据 | ✅ | 已有 The Odds API + ESPN |
| 球员级高阶统计 | ✅ | **FBref 已接入但只用到球队级**。FBref 免费提供球员级 xG/xA/渐进传球/抢断/拦截/争顶/过人，按场按位置。Understat 免费 API 补 xG/xGOT。这是球员评分的数据基石，**零成本可拿**。 |
| 战绩/交锋/积分 | ✅ | 已有 |
| 市场情绪/成交量 | ❌ | **双重否决**：① 真实投注量/成交量无免费源（需 Betfair Exchange 成交量 API，付费）；② 本项目 24 场回测已证"资金流向≈噪声、0/3 命中、ROI -100%"。**建议直接砍掉 Engine B，用"指数离散度"替代**（见下）。 |
| 24h 指数监控 | ✅ | 已有快照序列。异动捕获=对快照序列做阈值检测，纯工程。 |

> **200+ 维度**是营销数字。有效信号维度远小于此——你的回测已证明许多"维度"是噪声。**原则：宁缺毋滥，每个新维度必须过 paper_bets 复盘才上线。**

### 第 2 层 · 球员全息评分（8 算法）

| 评分 | 可行性 | 数据来源 |
|------|--------|----------|
| 进攻威胁（xG/xA/射门创造） | ✅ | FBref 球员表 |
| 防守压迫（抢断/拦截/争顶/反抢） | ✅ | FBref 球员表 |
| 影响力（终结+策动+受压传球） | ✅ | FBref progressive passes/carries |
| 即时战力（近 N 场×对手修正） | ✅ | FBref + 对手强度修正（可算） |
| 硬仗属性（评分 vs 对手强度） | ✅ | 上述评分 + 对手 xG 强度 |
| 稳定性（评分波动百分位） | ✅ | 评分序列方差 |
| 疲劳负荷（出场密度+旅行+休息） | ✅ | 赛程密度可算；旅行距离用球场坐标可算（你已有 venues 经纬度）；休息小时数=相邻比赛间隔 |
| 缺阵影响（在/离场净胜球差） | ⚠️ | FBref 无逐分钟事件；需 understat/Wyscout 级事件数据。**降级方案**：用"该球员在场时球队场均净胜球 vs 缺席场次"近似。 |

**"独家系数"** 是黑盒，但可从合理先验起步（按位置：前锋重 xG、后卫重拦截、中场重策动），再用你的 calibration 框架拟合系数——**这是你现有"由市场反推参数"方法论的直接延伸**。

### 第 3 层 · 市场量化

| 子项 | 可行性 | 说明 |
|------|--------|------|
| 趋势（早盘 vs 晚盘） | ✅ | 你已有快照序列，首尾帧对比即可 |
| 波动（指数变化速度） | ✅ | 快照序列的一阶差分 |
| 异常（多机构分歧/集体调降） | ✅ | 你已有 10 家公司快照，算离散度/方向一致性 |
| 阻力（高低位阻力） | ⚠️ | 概念模糊，可理解为"赔率在关键整数位的停留"——可做但收益存疑 |
| 首发预测 | ❌/⚠️ | **本方案最大数据瓶颈**。FBref/ESPN 不提供预测首发。可选：① api-football.com 付费有 predicted lineups（便宜档够用）；② FotMob 抓取（ToS 风险）；③ 退化为"注册名单 + 伤停过滤 + 上场首发"启发式。**MVP 先用方案③**。 |

> **替代 Engine B（市场成交量）**：用"10 家公司赔率离散度"作市场置信度代理——离散度低=市场共识高（你已在 `_build_basis` 算过两锚点 gap，扩展到全公司即可）。这把不可得的"成交量"换成可得的"共识度"，且无 ToS 风险。

### 第 4 层 · 预处理与风控

| 子项 | 可行性 | 说明 |
|------|--------|------|
| 伤停搜索网关 | ✅ | 已有懂球帝；可加 FBref/Transfermarkt 伤停交叉验证 |
| 教练战前发言 | ❌ | 需新闻 NLP，ROI 极低，**不建议做** |
| 裁判严苛度/偏向 | ⚠️ | Transfermarkt 有裁判档案（场均黄红牌、主场偏向）；Worldreferee 也可。可爬，**值得做**（杯赛裁判影响大）。 |
| 一票否决 | ✅ | 纯规则层：核心重伤/轮换触发降级标记 |

### 第 5 层 · 联赛分级

✅ **纯配置**。本项目当前聚焦世界杯单赛事，联赛分级暂不迫切；若扩展到俱乐部赛事，加一张 `league_tier` 表 + 权重映射即可。

### 第 6 层 · 五维引擎仲裁

| 引擎 | 可行性 | 落地 |
|------|--------|------|
| A 硬实力 | ✅ | 球员评分聚合 + 球队 xG + 积分 -> 理论胜负 |
| B 市场 | ⚠️→✅ | **砍掉成交量，改用赔率离散度**（见上） |
| C 情报（否决权） | ✅ | 伤停 + 赛程密度 -> 降级/否决标记 |
| D 概率 | ✅ | **复用现有泊松+Dixon-Coles**，加蒙特卡洛（对 score_matrix 采样 N 次）输出比分矩阵——本质你已有，蒙特卡洛是锦上添花 |
| E 仲裁 | ✅ | 规则优先级链：`if 否决触发: 降级; elif 联赛一档: 信 A; elif 二档: 信 B; else: 加权` |

> 球小策"5 个思维线程"的实质就是这个。**不必引入 LLM 做仲裁**——规则可解释、可回测、可复现，符合本项目"纯数学零主观"的文化。LLM 留给第 8 层的叙事生成。

### 第 7 层 · 战略分流（4 路径）

✅ **纯规则分类器**。输入=两队综合评分差 + 市场共识度 + 否决标记，输出 4 路径之一。逻辑文档已给清楚，直接编码。

### 第 8 层 · 策略输出

| 子项 | 可行性 | 说明 |
|------|--------|------|
| 单箭头/双选组合 | ✅ | 双选=两个互不独立的注，命中任一盈利。你的 EV 框架算的是单注；扩展到组合需要算联合概率——直接从 score_matrix 算（你已有比分矩阵）。 |
| 6 套策略组合 | ✅ | 都是 1X2/让球/大小的覆盖组合，给定概率即可计算 EV 与覆盖率 |
| 赛果推演（比分矩阵/半全场/进球数） | ✅ | 半全场=从 score_matrix 推半场分布（需半场 xG 模型，FBref 有半场数据）；进球数你已有 total_dist |
| 深度归因叙事 | ⚠️ | **唯一适合上 LLM 的地方**。你现在用规则模板生成中文解读（`_build_basis`/`_money_flow`），可升级为 LLM 生成"博弈逻辑"段落。但需固定 prompt + 事实约束，避免幻觉。 |

## 四、关键阻塞点与取舍

| 阻塞点 | 严重度 | 处置 |
|--------|--------|------|
| **市场成交量/情绪数据** | 高 | **砍掉**。无免费源 + 自家回测已证伪。用赔率离散度替代。 |
| **预测首发** | 高 | MVP 用"上场首发 + 伤停过滤"启发式；v2 接 api-football 付费档（约 $20/月够用）。 |
| **球员身价/评分（SofaScore/FotMob）** | 中 | 身价用 Transfermarkt（可爬，ToS 风险自担）；球员评分不依赖第三方，用 FBref 自算。 |
| **逐分钟事件（缺阵影响精确值）** | 中 | 降级为"在场/缺席场次场均净胜球差"近似。 |
| **裁判数据** | 低 | Transfermarkt 爬一次缓存（裁判风格稳定，低频更新）。 |
| **"独家系数"黑盒** | — | 不是阻塞——用先验起步 + calibration 拟合，正是你已在用的方法论。 |

**最大风险不是技术，是过拟合**：球员评分、裁判、疲劳这些维度加进来后，很容易在小样本上"看起来有效"实则方差。**必须复用现有 paper_bets + calibration 框架，每个新维度单独分账复盘，过不了 20 场门槛不上线**——和现在处理"顺资金策略"一样果断砍掉无效维度。

## 五、分阶段实施计划

### Phase 0 · 蒙特卡洛 + 策略组合（1-2 天，纯增量，零新数据）
**目标**：把现有概率引擎升级到球小策 D 引擎 + 第 8 层组合能力。

- [ ] `analyze.py` 加 `monte_carlo(mat, n=10000)`：对 score_matrix 采样，输出比分矩阵/半全场/任意盘口概率（验证与解析解一致）
- [ ] 扩展 `recs` 支持双选组合：算两注联合 EV（从 score_matrix 直接算，不独立假设）
- [ ] 实现 6 套策略组合的 EV + 覆盖率计算，进 HTML 报告
- [ ] paper_bets 增加组合注单结算逻辑

**价值**：纯利用现有数据，立即拿到球小策第 8 层 70% 的输出能力。**零额度成本、零新依赖**。

### Phase 1 · 球员级数据 + 8 项评分（1-2 周）
**目标**：补齐第 2 层，建立硬实力引擎（A）的数据底座。

- [ ] 新增 `sources/fbref_player.py`：抓球员级 matchlog（xG/xA/抢断/拦截/渐进传球/出场时间）
- [ ] 新增 `app/ratings.py`：8 套评分算法，按位置加权，输出 0-100；系数用先验 + calibration 拟合
- [ ] `db.py` 加 `player_stats` / `player_ratings` 表
- [ ] 球队综合评分 = 球员评分聚合（加权出场时间 + 伤停过滤）
- [ ] HTML 报告加"球员全息评分"卡（两队核心球员雷达图）

**验证门槛**：球员评分预测的胜负 vs 实际，logloss 要优于纯市场 1X2，否则不上线。

### Phase 2 · 仲裁引擎 + 战略分流（1 周）
**目标**：落地第 6/7 层。

- [ ] `app/engines.py`：A 硬实力 / B 离散度(替代成交量) / C 情报 / D 概率(复用) / E 仲裁
- [ ] `app/routing.py`：4 路径分类器
- [ ] 一票否决规则：核心球员缺阵影响值超阈值 -> 降级标记
- [ ] 仲裁结果与单 EV 推荐冲突时显性警示（沿用现有 `flow_note` 模式，不暗中加权）

**验证门槛**：仲裁推荐的 ROI 在 paper_bets 分账中需优于纯 EV 策略（当前亚盘 +11.4% 视作方差基线）。

### Phase 3 · 风控情报层（1-2 周，按需）
**目标**：补齐第 4 层，提升杯赛/淘汰赛准确度。

- [ ] `sources/transfermarkt.py`：裁判档案（场均黄红牌、主场偏向），爬一次缓存
- [ ] 伤停交叉验证：懂球帝 + FBref + Transfermarkt 三源校验
- [ ] 疲劳负荷评分：赛程密度 + 旅行距离（用 venues 经纬度算）+ 休息小时数
- [ ] 首发预测 v1：上场首发 + 伤停过滤 + 轮换风险（72h 内有比赛则标记）

### Phase 4 · LLM 叙事生成（可选，1 周）
**目标**：升级第 8 层"深度归因"文字。

- [ ] 用 Claude API 生成"博弈逻辑"段落：输入=结构化事实（评分差/伤停/盘口走势/路径判定），输出=自然语言归因
- [ ] 严格事实约束：prompt 只允许引用传入的事实字段，禁用外部知识，降低幻觉
- [ ] 规则模板作为 fallback（LLM 失败时不影响报告生成）

> 不建议用 LLM 做仲裁或选注——不可解释、不可回测。LLM 只做"把结构化结论讲给人听"。

## 六、建议的最小可行方案（MVP）

若资源有限，**只做 Phase 0 + Phase 1 的球员评分子集**即可获得最大边际收益：

1. **蒙特卡洛 + 双选组合**（Phase 0）——立即有组合策略输出，零成本
2. **FBref 球员级 xG/评分**（Phase 1 核心）——这是你唯一还没用的免费金矿，直接补齐"赔率之外的信息层"
3. **赔率离散度替代成交量**——把现有的 10 家快照从"走势图"升级为"市场共识度"指标
4. **严守 paper_bets 复盘**——每个新维度分账，过不了门槛就砍

**不做**：市场成交量（已证伪）、教练发言 NLP（ROI 低）、追逐 200+ 维度（营销陷阱）、LLM 仲裁（不可回测）。

## 七、技术选型

| 组件 | 选型 | 理由 |
|------|------|------|
| 球员数据 | FBref + Understat | 免费、覆盖全、已在用 |
| 裁判/身价 | Transfermarkt | 免费（爬取，注意 ToS） |
| 预测首发(可选) | api-football 付费档 | 便宜、稳定、有 API |
| 蒙特卡洛 | 标准库 `random` | 对 score_matrix 采样，无需新依赖 |
| 评分拟合 | 复用 `fit_model` 网格搜索思路 | 与现有方法论一致 |
| 仲裁/分流 | 纯 Python 规则 | 可解释、可回测 |
| 叙事(可选) | Claude API | 仅生成文字，事实约束 |
| 存储 | 扩展现有 SQLite | 加 `player_stats`/`player_ratings`/`referees` 表 |
| 调度 | 复用 GitHub Actions | 球员数据低频，加到 scrape.yml 即可 |
| 校验 | 复用 `calibration.json` + `paper_bets` | 一切新维度必须过这关 |

## 八、风险与原则

1. **过拟合是头号风险**。每加一个维度，paper_bets 里单开一个 `strategy` 分账（就像现在的 `ev`/`flow`），20 场内 ROI 不显著就砍——别舍不得。
2. **可解释优先于"智能"**。球小策的"5 线程仲裁"听起来高级，但规则链 `信情报 > 信市场 > 信实力` 才是本质。保持规则可解释，拒绝黑盒加权。
3. **数据 ToS 风险**。FBref/Understat 遵守 robots（你已在做）；Transfermarkt/FotMob 爬取属灰色地带，低频 + 缓存 + UA 礼貌，杯赛/小规模可接受，商业化前需评估。
4. **不追营销指标**。"200+ 维度""第三代全息""独家系数"都是包装。你的 98 场回测比任何营销话术都可靠——继续用数据说话。
5. **保持纯数学文化**。新增的信息层（评分/裁判/疲劳）应作为**模型输入的修正项或独立仲裁因子**，而非主观调参。能被 calibration 拟合的才上线。

---

## 九、结论

**球小策架构整体可行**，且你已有约 40% 的底座（数据管道、概率模型、校准复盘、自动发布）。剩余 60% 中：

- **约 30% 是纯规则层**（联赛分级/战略分流/策略组合/仲裁），工程量小、可解释、可回测——值得做。
- **约 20% 是免费数据金矿**（FBref 球员级 xG/评分、赔率离散度、裁判数据）——这是寻找真实 edge 的最现实路径，**优先做**。
- **约 10% 是阻塞/已证伪**（市场成交量、教练发言）——**明确不做**。

**核心判断**：方向正确（edge 在信息层），路径清晰（复用现有回测框架逐维验证），最大风险是过拟合而非技术。建议从 **Phase 0（蒙特卡洛+组合，零成本）** 起步，再用 **Phase 1（FBref 球员评分）** 检验"信息层是否真能榨出统计显著的 edge"——这一步的结果决定后续是否值得继续投入。
